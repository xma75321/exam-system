## 1. 测试策略总览

### 1.1 测试层级

| 层级 | 类型 | 覆盖内容 |
|------|------|----------|
| L1 | 接口测试（API） | 所有 REST 接口的入参、出参、状态码、错误码 |
| L2 | 业务流程测试（E2E） | 端到端用户场景：注册→上传→组卷→答题→评分→报告 |
| L3 | 前端 UI 测试 | 页面渲染、交互、表单验证、响应式布局 |
| L4 | 非功能测试 | 性能、安全、可靠性、兼容性 |

### 1.2 测试优先级

- **P0 用例**：覆盖所有 P0 功能需求，必须 100% 通过方可发布
- **P1 用例**：覆盖 P1 功能需求，阻塞性缺陷需修复后发布
- **P2 用例**：覆盖非功能需求及边界/异常场景
