# 智能考试系统（MVP）— Sprint 计划

## 总览

| Sprint | 周期 | 目标 | 任务范围 |
|--------|------|------|----------|
| Sprint 1 | 第 1-2 周 | 基础设施 + 核心后端 + 解析引擎 | TASK 001-015 |
| Sprint 2 | 第 3-4 周 | 前端页面 + 答题评分 + 报告部署 | TASK 016-028 |

## Sprint 1：基础 + 核心后端（第 1-2 周）

### 目标
完成项目初始化、数据库设计、用户认证、Word 解析引擎、题库管理和考试管理 API。

### Sprint 1 里程碑
- [ ] 后端项目可启动（FastAPI + MySQL 连接）
- [ ] 用户可注册、登录
- [ ] 可上传 Word 文件并解析为结构化题目
- [ ] 题目可存入数据库并通过 API 查询
- [ ] 可通过 API 创建和管理考试

- Day 5：TASK 007（文件上传 API）

**第 2 周**：

### Sprint 1 验收标准
1. `curl POST /api/auth/register` 可创建用户
2. `curl POST /api/auth/login` 可获取 JWT Token
3. `curl POST /api/upload -F file=test.docx` 可返回解析后的题目 JSON
4. `curl POST /api/upload/confirm` 可将题目存入数据库
5. `curl GET /api/questions` 可查询题目列表
6. `curl POST /api/exams` 可创建考试

5. 提交后自动评分，跳转到结果页
6. 结果页显示分数、正确率、逐题回顾
7. `/reports` 显示统计概览和趋势图

## 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| Word 格式多样，解析准确率低 | 用户体验差 | 定义严格的格式规范，提供格式示例模板 |
| MySQL 连接问题 | 后端无法启动 | 提供 Docker Compose 配置 |
| 前后端联调耗时 | 延期 | API 先行，前端 Mock 数据开发 |
| 简答题评分接口复杂 | MVP 范围扩大 | MVP 仅预留接口，不实现自动评分 |
