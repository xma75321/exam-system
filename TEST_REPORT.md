## 发现的问题

### 问题1: 题目类型定义不一致 [已修复]

**严重程度**: 中  
**描述**: 数据库 `questions.type` 字段使用枚举类型 `enum('single','multi','judge','fill','essay')`，但测试代码中使用了 `choice` 作为选择题类型。  
**影响**: 导致题目筛选失败  
**修复方案**: 
1. 测试脚本使用 `single` 而不是 `choice`
2. 前端和后端已统一使用 `single/multi/judge/fill/essay`

### 问题2: 注册后无法立即登录 [已修复]

**严重程度**: 高  
**描述**: 用户在前端注册成功后，无法立即登录，提示"用户名或密码错误"。  
**修复方案**: 
1. 后端注册API已正确提交数据库事务
2. 前端注册成功后显示"注册成功，请登录"提示
3. 自动切换到登录标签页并填充用户名

### 问题3: 测试数据清理不彻底 [已修复]

**严重程度**: 低  
**描述**: 测试脚本中的清理函数未能完全删除测试数据。  
**修复方案**: 按正确顺序删除（先删子表再删父表）

## API 接口清单

### 认证接口
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/me` - 获取当前用户

### 题目接口
- `GET /api/questions` - 题目列表（支持分页和筛选）
- `GET /api/questions/{id}` - 题目详情
- `DELETE /api/questions/{id}` - 删除题目

### 试卷接口
- `POST /api/upload/confirm` - 保存解析后的题目

### 考试接口
- `POST /api/exams` - 创建考试
- `GET /api/exams` - 考试列表
- `POST /api/exams/{id}/publish` - 发布考试
- `POST /api/exams/{id}/close` - 关闭考试
- `DELETE /api/exams/{id}` - 删除考试

### 答题接口
- `POST /api/attempts` - 开始考试
- `GET /api/attempts/{id}` - 获取答题进度
- `PUT /api/attempts/{id}/answers` - 保存答案
- `POST /api/attempts/{id}/submit` - 提交考试
- `GET /api/attempts/{id}/result` - 获取考试结果

### 统计接口
- `GET /api/reports/overview` - 统计总览
- `GET /api/reports/trend` - 成绩趋势
